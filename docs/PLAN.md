# Habitris 개발 계획 (Plan)

기획서([PROJECT_SPEC.md](PROJECT_SPEC.md))를 기준으로 한 단계별 개발 계획이다.

---

## Phase 0: 준비

| 항목 | 내용 |
|------|------|
| 스택 선정 | PC 앱: Electron + React 또는 Tauri + React 등 |
| 저장소·환경 | Git 초기화, dev/prod 환경 분리 |
| 디자인 톤 | 미니멀 UI 톤·색감·테트리스 블록 스타일 가이드 |

---

## Phase 1: PC 앱 — 핵심 기능

### 1.1 습관 CRUD + 데이터 구조
- 습관 추가·수정·삭제·목록
- 습관 ↔ 테트리스 블록 타입/색 매핑
- 로컬 저장(예: localStorage 또는 로컬 DB)으로 상태 유지

### 1.2 테트리스 필드
- 고전 테트리스 그리드(예: 10×20)
- 습관 완료 시 해당 블록이 **자동으로** 다음 낙하 위치에 쌓이기
- **직접 조작**: 블록 드래그·회전·이동으로 배치
- 한 줄이 꽉 차면 줄 제거 + 그 줄을 Castle로 전달하는 로직

### 1.3 Castle 페이지
- 완성된 줄들이 “성”처럼 아래부터 쌓여서 표시
- 히스토리(날짜/횟수) 간단 조회 가능하면 가산

### 1.4 기본 네비게이션
- 습관·테트리스 메인 ↔ Castle 전환
- 설정(나중에 광고·테마 등) 진입점만 준비

---

## Phase 2: PC 앱 — 완성도

| 항목 | 내용 |
|------|------|
| UX | 애니메이션(블록 낙하·줄 제거·Castle 쌓임), 피드백 메시지 |
| 반응형 | 창 크기 변경 시 레이아웃 깨지지 않도록 |
| 데이터 | 백업/복원 또는 계정 연동 여부 결정 후 구현 |
| 테스트 | 핵심 플로우(습관 완료 → 줄 완성 → Castle) 자동/수동 테스트 |

---

## Phase 3: 수익화 (광고)

- 광고 SDK 선정(예: Electron용 또는 웹뷰 기반)
- 노출 위치: **메인 플레이 영역 제외**, 설정·여백·전환 시 등만 사용
- 노출 빈도·스킵 옵션 등 사용성 기준으로 조정

---

## Phase 4: 모바일 앱 + Play Store

| 항목 | 내용 |
|------|------|
| 모바일 스택 | React Native 또는 Flutter 등, PC와 로직 공유 가능한 구조 선정 |
| 터치 UI | 습관 탭·테트리스 드래그/회전을 터치에 맞게 조정 |
| 광고 | 모바일 광고 SDK(예: AdMob) 연동, 정책·배치 동일 원칙 |
| 스토어 | 패키지명·스크린샷·설명·개인정보처리방침 준비 후 Play Store 배포 |
| 배포 | 무료 다운로드, 인앱 광고만 수익화 유지 |

---

## 마일스톤 요약

| 단계 | 목표 |
|------|------|
| **M1** | Phase 0 완료 + 습관 CRUD 동작 |
| **M2** | 테트리스 자동/수동 쌓기 + 줄 완성 → Castle 반영 |
| **M3** | PC 앱 사용 가능한 수준(Phase 2) |
| **M4** | 광고 연동 + 사용성 유지 |
| **M5** | 모바일 앱 출시 + Play Store 배포 |

# 코드 리뷰 예시

## 예시 1: 피드백 구분

**코드**: API 키가 클라이언트 번들에 하드코딩됨

- **🔴 필수**: `src/config.ts` — API 키를 환경 변수로 분리하고, 클라이언트 번들에 포함되지 않도록 빌드 설정 확인
- **🟡 제안**: 에러 메시지를 사용자 친화적으로 바꾸고, 개발 시에만 상세 로그 노출
- **🟢 참고**: 나중에 키 로테이션 정책을 문서에 추가하면 좋음

## 예시 2: Habitris 용어 점검

**코드**: 주석에 "라인 클리어"라고만 적혀 있음

- **🟡 제안**: Habitris 용어에 맞게 "줄 완성 → Castle 기록" 맥락을 주석에 추가하면 기획서와 일치해 유지보수에 유리함

## 예시 3: 요약 문장

- "전반적으로 로직은 명확하고, 보안 이슈 1건(필수)과 네이밍·주석 개선 제안 2건이 있습니다."
- "Habitris 용어와 기획서와 일치하며, 테스트만 한 파일 추가하면 좋겠습니다."

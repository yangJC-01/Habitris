# 코드 리뷰 시 참고할 스타일·표준

리뷰 시 아래 기준으로 일관성을 확인할 때 참고한다. 프로젝트에 별도 스타일가이드가 있으면 해당 문서를 우선한다.

## 일반

- **네이밍**: 변수·함수는 역할이 드러나게, 상수는 대문자/스네이크 등 언어 관례 준수
- **함수 크기**: 한 함수는 한 가지 일에 집중, 지나치게 길면 분리 검토
- **중복**: 동일 로직 반복 시 추출·공통화 여부 검토
- **에러 처리**: 예외·에러 경로가 명시되어 있고, 사용자/로그에 적절히 전달되는지

## Habitris

- **용어**: 습관 완료 → 블록 쌓기, 줄 완성 → Castle 기록 (문서·코드·주석 통일)
- **문서**: docs/PROJECT_SPEC.md, .cursor/rules/habitris-conventions.mdc와 충돌 없는지

## 프론트/백 구분 시

- **접근성**: 버튼·폼 등에 의미 있는 라벨·역할
- **상태 관리**: 예측 가능한 단일 소스, 불필요한 전역 상태 없는지
- **API·데이터**: 응답/에러 처리, 타임아웃·재시도 등 고려 여부

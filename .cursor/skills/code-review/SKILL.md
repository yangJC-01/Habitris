# 코드 리뷰 에이전트

기존 코드 또는 변경된 코드를 일관된 기준으로 검토할 때 이 스킬을 따른다.

## 적용 시점

- 사용자가 "코드 리뷰 해줘", "이 부분 검토해줘" 등으로 요청할 때
- PR/변경분 검토를 요청할 때
- 리팩터링·기능 추가 전 현재 코드 상태를 점검할 때

## 리뷰 체크리스트

리뷰 시 아래를 순서대로 확인한다.

- [ ] **로직·정확성**: 동작이 의도대로인지, 엣지 케이스·예외 처리 여부
- [ ] **보안**: SQL 인젝션, XSS, 민감정보 노출, 권한 검증 등
- [ ] **가독성·유지보수성**: 함수/모듈 크기, 네이밍, 중복, 의존성
- [ ] **테스트**: 변경/추가된 부분에 대한 테스트 존재·충분성
- [ ] **프로젝트 규칙**: Habitris 컨벤션·기획서와의 일치 여부

## Habitris 프로젝트 특화

이 리포지토리는 Habitris이므로 다음을 함께 확인한다.

- **용어**: 습관 완료 → 블록 쌓기, 줄 완성 → Castle 기록 등 docs/PROJECT_SPEC.md·habitris-conventions와 일치하는지
- **기획서 정합성**: docs/PROJECT_SPEC.md 기준으로 기능·플랫폼·수익 모델과 맞는지
- **플랫폼 순서**: 1차 PC 앱 → 2차 모바일(Play Store) 전제가 반영되었는지

도메인 용어·플로우는 `.cursor/skills/habitris-domain/SKILL.md` 참고.

## 피드백 형식

리뷰 결과는 아래 형식으로 구분해 제시한다.

- **🔴 필수**: 머지/배포 전 반드시 수정
- **🟡 제안**: 개선을 권장하는 사항
- **🟢 참고**: 선택적 개선 또는 추가 정보

각 항목에는 **위치(파일/라인 또는 범위)** 와 **이유·수정 방향**을 함께 적는다.

## 출력 템플릿

```markdown
## 코드 리뷰 요약
[한 줄 요약]

### 🔴 필수
- (없으면 "없음"으로 명시)

### 🟡 제안
- …

### 🟢 참고
- …

### 프로젝트 규칙 점검 (Habitris)
- 용어·기획서 일치: …
- 플랫폼·기능 정합성: …
```
